## Introduction

Chat ‚Äì a simple word for a simple app. No nuclear collider of pointless, useless features here. As a developer who thinks WordPress is shit and would rather build something fresh from the ground several times, I wanted an app that‚Äôs as straightforward as its name.

C ‚Äì Cross-platform:
I'm not a tech-giant to have time and money to batter with one button looking the same on different devices and platforms. So... we have choice: React-Native. That is not just a web-view (based on MTAA requirements), JS ‚ù§Ô∏è, and good, excellent ecosystem.

H ‚Äì Huge Performance:
I'm a programmer, and if my system doesn‚Äôt boot in a second, it‚Äôs bullshit. Sure, React Native might throw a wrench in the works sometimes, but in good hands and with a little clever tinkering it‚Äôs fast as hell (don't know why hell is fast). And on the backend, we‚Äôre using Bun to keep message sending lightning quick.

A ‚Äì Atomicity:
When millions of messages are flying around, reliability is non-negotiable. With Bun and Elysia (plus their awesome plugins), every message is handled solidly, with logs and stats that actually mean something.

T ‚Äì Touch-Friendly:
I don't like when mobile application are designed like I am using the phone with two hands. **I CANNOT CLICK ON LEFT TOP CORNER BUTTON**. That is even better on iphoneos that does not support gestures, and you need to touch in that corner several times.

So... Simple, Fast, Atomic, and (most importantly) Easy to use.

## Project Overview

### Pointed platforms

#### Front-end

Based on possibilities of React Native, we can build applications for iOS, **Android**, and **Web**. Due to limitation of non-using and negative experience about iOS/iPhones we are not sure about shipping iOS build.

So primary we are focusing on stable Android and Web builds, with optional iOS.

#### Back-end

The back-end is built on Elysia, which is a TypeScript framework supercharged by Bun. With possibilities of Bun - it can be built to binary and run on every machine.

Primary, production target is **Linux**, and developing include **Windows**.

```mermaid
flowchart LR
```

Optional Two-Factor Authentication:

```mermaid
flowchart LR
    S8[Backend returns authentication success]
```

##### Chat message sending

```mermaid
flowchart LR
  %% Trigger event: User selects a chat
  TRIGGER([User selects a chat]) --> CHOICE;
  style TRIGGER fill:#ccf

  %% User chooses action: text or attachment
  CHOICE[User chooses action:<br>- Enter text message<br>- Click attachment button] --> DECIDE;

  %% Decision: is a text message entered?
  DECIDE{Did the user enter a text message?}

  %% Text Message Flow
  DECIDE -- Yes --> SENDTEXT["User clicks 'Send' for text message"]
  SENDTEXT --> LOCALSYNC[Text message saved in local storage]
  LOCALSYNC --> SENDSERVER[Text message sent to server]
  SENDSERVER --> CHECKTEXT{Was text message sent successfully?}
  CHECKTEXT -- Yes --> TEXTSUCCESS[Text message delivered]
  CHECKTEXT -- No --> RETRYTEXT[Prompt user to retry sending text message]

  %% Attachment Flow
  DECIDE -- No --> ATTACHFLOW[User clicks attachment button]
  ATTACHFLOW --> CHOOSEATTACH[User selects attachment type]

  %% Attachment Options: Camera, Gallery, or Geo-location
  CHOOSEATTACH --> CAMERA[Select image from camera]
  CHOOSEATTACH --> GALLERY[Select image from gallery]
  CHOOSEATTACH --> GEO[Select current geo location]

  %% Preview before sending attachment
  CAMERA --> PREVIEWCAM[Preview camera image]
  GALLERY --> PREVIEWGAL[Preview gallery image]
  GEO --> PREVIEWGEO[Preview geo location]

  %% All previews lead to sending attachment
  PREVIEWCAM --> SENDATTACH;
  PREVIEWGAL --> SENDATTACH;
  PREVIEWGEO --> SENDATTACH;

  SENDATTACH["User clicks 'Send' for attachment"]
      --> LOCALSYNC2[Attachment synced in local storage]
      --> SENDSERVER2[Attachment sent to server]
      --> CHECKATTACH{Was attachment sent successfully?}
  CHECKATTACH -- Yes --> ATTACHSUCCESS[Attachment delivered]
  CHECKATTACH -- No --> RETRYATTACH[Prompt user to retry sending attachment]
```

### UATs

## Positive Tests

### Test 1: Create New Account

**Input Conditions**
- User has an internet connection.
- User provides a valid email, username, and password.
- Password meets security requirements (e.g., length, special characters).

**Output Conditions**
- User receives a success message.
- User is redirected to the chat list page.

**Steps**
1. Open the app and navigate to the registration page.
2. Enter a valid email, username, and password.
3. Click the "Continue" button.
4. Add profile description and avatar.
5. Click "Sign Up".

**Result**: PASS/FAIL

---

### Test 2: Send Message with Geolocation

**Input Conditions**
- User has an internet connection.
- User is logged in.
- User has a chat with another user.

**Output Conditions**
- Message with geolocation is sent.

**Steps**
1. Open the app and navigate to an existing chat.
2. Click the "Attachments" button.
3. Click the "Geolocation" button.
4. Allow location permissions if prompted.
5. Click the "Send" button.
